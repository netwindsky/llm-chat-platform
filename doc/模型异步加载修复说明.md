# 模型异步加载修复说明

## 问题

之前模型切换时出现死锁和 503 错误，原因是：
1. 加载模型时等待 llama-server 完全启动才返回（耗时几十秒）
2. 前端不知道模型加载进度，立即发送聊天请求
3. llama-server 还没启动完成，返回 503 错误

## 解决方案

采用**异步加载 + 前端轮询**的模式：

### 流程

```
1. 前端调用加载模型 API
   ↓
2. 后端立即创建进程并返回 success（不等待）
   ↓
3. 后端后台任务异步等待 llama-server 启动
   ↓
4. 前端轮询模型状态（每秒一次）
   ↓
5. 状态变为 loaded 后，前端允许发送消息
   ↓
6. 聊天请求成功
```

### 状态流转

```
loading → loaded (成功)
loading → error (失败)
```

## 关键代码

### 后端 - llama_backend.py

```python
async def initialize(self, model_path: str, model_config: Dict[str, Any]) -> bool:
    """初始化模型（异步，立即返回）"""
    # 启动 llama-server 进程
    self._process = subprocess.Popen(cmd, ...)
    
    # 设置初始状态为 loading
    self._model_info = ModelInfo(status=ModelStatus.LOADING)
    
    # 创建后台任务等待服务器启动
    asyncio.create_task(self._wait_and_set_status())
    
    # 立即返回
    return True

async def _wait_and_set_status(self):
    """后台等待服务器启动并更新状态"""
    await self._wait_for_server(120)
    # 更新状态为 loaded
    self._model_info.status = ModelStatus.LOADED
```

### 前端 - models.js

```javascript
async function loadModel(modelId, config = {}) {
  const res = await modelApi.load(modelId, config)
  if (res.success) {
    // 轮询等待模型完全加载
    await waitForModelLoaded(modelId)
  }
}

async function waitForModelLoaded(modelId, maxAttempts = 120, interval = 1000) {
  for (let i = 0; i < maxAttempts; i++) {
    await new Promise(resolve => setTimeout(resolve, interval))
    const model = await modelApi.get(modelId)
    if (model.status === 'loaded') {
      return true
    }
  }
  throw new Error('Loading timeout')
}
```

## 优势

1. **快速响应**：加载请求立即返回，用户体验好
2. **准确状态**：前端实时了解模型加载进度
3. **避免错误**：只在 loaded 状态才允许聊天
4. **资源管理**：自动卸载旧模型，避免显存占用

## 注意事项

1. 轮询超时时间设置为 120 秒，可根据模型大小调整
2. 轮询间隔 1 秒，避免请求过于频繁
3. 后端使用后台任务异步启动，确保不会阻塞主线程
