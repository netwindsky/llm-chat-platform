# Llama-Server 空闲监控功能

## 功能概述

空闲监控功能可以自动监控 llama-server 的请求活动，当某个端口的服务在指定时间内没有收到请求时，自动卸载该端口加载的模型，释放内存资源。

## 主要特性

- ✅ **自动监控**: 自动跟踪每个 llama-server 端口的最后请求时间
- ✅ **可配置超时**: 支持全局默认超时和按端口单独配置
- ✅ **开关控制**: 可以全局启用/禁用，也可以按端口单独控制
- ✅ **API 管理**: 提供 REST API 查看状态、修改配置、手动重置计时器
- ✅ **实时反馈**: 空闲超时时自动卸载并记录日志

## 配置说明

### 1. 全局配置 (configs/server.yaml)

```yaml
# 空闲监控配置（自动卸载空闲模型）
idle_monitor:
  enabled: true                    # 是否启用空闲监控
  timeout: 60                      # 默认空闲超时时间（秒），默认1分钟
  check_interval: 10               # 检查间隔（秒）
  
  # 按端口单独配置（可选）
  port_configs:
    # 端口 38521 使用2分钟超时
    38521:
      enabled: true
      timeout: 120
    
    # 端口 38522 禁用空闲监控
    38522:
      enabled: false
```

### 2. 模型配置 (configs/models.yaml)

可以在单个模型的配置中覆盖全局设置：

```yaml
- id: qwen3.5-9b-ud-q4
  name: Qwen3.5-9B-UD-Q4
  idle_monitor_enabled: true    # 启用监控
  idle_timeout: 120             # 2分钟超时
  port: 38521
  
- id: another-model
  name: Another Model
  idle_monitor_enabled: false   # 禁用监控
  port: 38522
```

## API 接口

### 模型卸载接口

#### 1. 卸载指定模型

```http
POST /api/v1/models/{model_id}/unload
```

**响应示例：**
```json
{
  "success": true,
  "message": "Model qwen3.5-9b-ud-q4 unloaded successfully"
}
```

#### 2. 卸载所有模型（快捷接口）

```http
GET /unload
```

**说明：** 使用 GET 方法，无需请求体，直接在浏览器或命令行访问即可。

**响应示例：**
```json
{
  "success": true,
  "message": "Unloaded 3 models",
  "unloaded_models": ["model1", "model2", "model3"],
  "failed_models": []
}
```

#### 3. 卸载所有模型（标准接口）

```http
POST /api/v1/models/unload
```

**说明：** 功能与快捷接口相同，路径不同。

#### 4. 按端口卸载模型

```http
POST /api/v1/models/unload-by-port/{port}
```

**响应示例：**
```json
{
  "success": true,
  "message": "Model qwen3.5-9b-ud-q4 on port 38521 unloaded successfully",
  "model_id": "qwen3.5-9b-ud-q4",
  "port": 38521
}
```

### 空闲监控接口

#### 4. 获取所有监控状态

```http
GET /api/v1/models/idle-monitor/status
```

**响应示例：**
```json
{
  "success": true,
  "monitors": {
    "38521": {
      "port": 38521,
      "enabled": true,
      "idle_timeout": 60,
      "model_id": "qwen3.5-9b-ud-q4",
      "last_request_time": 1709876543.123,
      "idle_time": 45.6,
      "is_idle": false,
      "monitoring": true
    }
  }
}
```。

#### 5. 获取指定端口状态

```http
GET /api/v1/models/idle-monitor/{port}/status
```

**响应示例：**
```json
{
  "port": 38521,
  "enabled": true,
  "idle_timeout": 60,
  "model_id": "qwen3.5-9b-ud-q4",
  "idle_time": 45.6,
  "is_idle": false,
  "monitoring": true
}
```

#### 6. 更新端口配置

```http
POST /api/v1/models/idle-monitor/{port}/config
Content-Type: application/json

{
  "enabled": true,
  "timeout": 120
}
```

**响应示例：**
```json
{
  "success": true,
  "message": "Idle monitor config updated for port 38521",
  "config": {
    "enabled": true,
    "timeout": 120
  }
}
```

#### 7. 重置空闲计时器

```http
POST /api/v1/models/idle-monitor/{port}/reset
```

**响应示例：**
```json
{
  "success": true,
  "message": "Idle timer reset for port 38521"
}
```

## 使用示例

### 场景1: 测试空闲监控

```bash
# 1. 加载模型
curl -X POST http://localhost:8080/api/v1/models/qwen3.5-9b-ud-q4/load

# 2. 查看监控状态
curl http://localhost:8080/api/v1/models/idle-monitor/38521/status

# 3. 等待60秒（不发送请求）

# 4. 模型自动卸载，查看日志：
# [IdleMonitor] 端口 38521 空闲 60.0秒，超过阈值 60秒
# [IdleMonitor] 成功卸载模型 qwen3.5-9b-ud-q4
```

### 场景2: 动态调整超时时间

```bash
# 将端口 38521 的超时时间改为5分钟
curl -X POST http://localhost:8080/api/v1/models/idle-monitor/38521/config \
  -H "Content-Type: application/json" \
  -d '{"enabled": true, "timeout": 300}'
```

### 场景3: 临时禁用监控

```bash
# 禁用端口 38521 的空闲监控
curl -X POST http://localhost:8080/api/v1/models/idle-monitor/38521/config \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
```

### 场景4: 手动卸载模型

```bash
# 卸载指定模型
curl -X POST http://localhost:8080/api/v1/models/qwen3.5-9b-ud-q4/unload

# 卸载所有模型（快捷接口）
curl http://localhost:8080/unload

# 或者在浏览器中直接访问
# http://localhost:8080/unload

# 卸载所有模型（标准接口）
curl -X POST http://localhost:8080/api/v1/models/unload

# 按端口卸载模型（卸载端口 38521 上的模型）
curl -X POST http://localhost:8080/api/v1/models/unload-by-port/38521
```

## 日志输出

启用监控后，系统会输出以下日志：

```
[IdleMonitor] 开始监控端口 38521，空闲超时: 60秒
[IdleMonitor] 端口 38521 空闲 60.0秒，超过阈值 60秒
[IdleMonitor] 端口 38521 的模型 qwen3.5-9b-ud-q4 空闲超时，准备卸载
[IdleMonitor] 成功卸载模型 qwen3.5-9b-ud-q4
[IdleMonitor] 停止监控端口 38521
```

## 注意事项

1. **默认启用**: 新功能默认启用，超时时间为60秒
2. **向后兼容**: 如果不配置，使用默认设置，不影响现有功能
3. **多端口支持**: 每个端口独立监控，互不影响
4. **请求触发**: 只有调用 chat 或 chat_stream 时才会更新最后请求时间
5. **手动控制**: 可以通过 API 随时修改配置或重置计时器

## 故障排查

### 模型没有自动卸载

1. 检查监控是否启用：`GET /api/v1/models/idle-monitor/{port}/status`
2. 检查空闲时间是否达到阈值
3. 查看服务器日志是否有错误信息

### 误卸载模型

1. 增加超时时间：`POST /api/v1/models/idle-monitor/{port}/config`
2. 或禁用该端口的监控：`{"enabled": false}`

### API 返回 404

确保端口已被监控（需要先加载模型）
