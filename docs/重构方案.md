# LLM 推理平台重构方案

## 📋 项目概述

### 当前问题
1. ❌ 目录结构混乱，模型文件散落各处
2. ❌ 缺少统一的模型管理
3. ❌ 没有友好的 Web 界面
4. ❌ API 服务分散，难以维护
5. ❌ 不支持多推理后端切换（llama、sglang 等）
6. ❌ 缺少模型分类和元数据管理

### 重构目标
1. ✅ 清晰的目录结构和项目组织
2. ✅ 统一的模型管理中心
3. ✅ 现代化的 Web 聊天界面
4. ✅ 统一的 API 网关服务
5. ✅ 可插拔的推理后端架构
6. ✅ 完善的模型分类和标签系统

```
├─────────────────────────────────────────┤

├─────────────────────────────────────────┤
│  Llama  │  SGLang  │  vLLM  │  Others  │  ← 推理实现层
└─────────────────────────────────────────┘
```

### 2. 后端抽象接口

```python
# server/core/backend.py
from abc import ABC, abstractmethod
from typing import AsyncGenerator, Dict, Any

class InferenceBackend(ABC):
    """推理后端抽象基类"""
    
    @abstractmethod
    async def initialize(self, model_path: str, config: Dict[str, Any]) -> bool:
        """初始化模型"""
        pass
    
    @abstractmethod
    async def generate(self, prompt: str, config: Dict[str, Any]) -> AsyncGenerator[str, None]:
        """流式生成"""
        pass
    
    @abstractmethod
    async def chat(self, messages: list, config: Dict[str, Any]) -> Dict[str, Any]:
        """对话生成"""
        pass
    
    @abstractmethod
    async def get_model_info(self) -> Dict[str, Any]:
        """获取模型信息"""
        pass
    
    @abstractmethod
    async def shutdown(self) -> None:
        """关闭后端"""
        pass
    
    @property
    @abstractmethod
    def name(self) -> str:
        """后端名称"""
        pass
    
    @property
    @abstractmethod
    def is_available(self) -> bool:
        """是否可用"""
        pass
```

### 3. 后端工厂模式

```python
# server/backends/backend_factory.py

class BackendFactory:
    """后端工厂"""
    
    _backends = {
        'llama': LlamaBackend,
        'sglang': SGLangBackend,
        'vllm': VLLMBackend,
    }
    
    @classmethod
    def create(cls, backend_type: str, config: Dict[str, Any]) -> InferenceBackend:
        """创建后端实例"""
        if backend_type not in cls._backends:
            raise ValueError(f"Unsupported backend: {backend_type}")
        
        return cls._backends[backend_type](config)
    
    @classmethod
    def list_available(cls) -> list:
        """列出可用的后端"""
        available = []
        for name, backend_class in cls._backends.items():
            if backend_class.is_available:
                available.append(name)
        return available
```

## 🔌 API 接口设计

### 1. RESTful API 规范

#### 模型管理
```http
GET    /api/v1/models              # 获取模型列表
GET    /api/v1/models/:id          # 获取模型详情
POST   /api/v1/models/:id/load     # 加载模型
POST   /api/v1/models/:id/unload   # 卸载模型
DELETE /api/v1/models/:id          # 删除模型
```

#### 聊天接口
```http
POST   /api/v1/chat/completions    # 聊天完成（OpenAI 兼容）
POST   /api/v1/chat/stream         # 流式聊天
GET    /api/v1/chat/history/:id    # 获取聊天历史
DELETE /api/v1/chat/history/:id    # 删除聊天历史
```

#### 系统接口
```http
GET    /api/v1/health              # 健康检查
GET    /api/v1/stats               # 系统统计
GET    /api/v1/backends            # 获取可用后端
POST   /api/v1/backends/switch     # 切换后端
```

```python
# 预加载热门模型
class ModelPool:
    def __init__(self, max_models=3):
        self.pool = {}
        self.max_models = max_models
    
    async def get(self, model_id: str) -> InferenceBackend:
        if model_id not in self.pool:
            if len(self.pool) >= self.max_models:
                # LRU 淘汰
                await self._evict()
            # 加载新模型
            backend = await self._load_model(model_id)
            self.pool[model_id] = backend
        return self.pool[model_id]
```
