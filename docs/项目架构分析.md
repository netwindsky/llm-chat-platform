# LLM 推理平台项目架构分析

## 1. 整体架构

这是一个基于 FastAPI 的 LLM 推理平台，采用分层架构：

```
┌─────────────────────────────────────────────────────────────┐
│  API 层 (server/api/v1/)                                     │
│  ├── chat.py      - 聊天接口 (/api/v1/chat/completions)     │
│  ├── openai.py    - OpenAI 兼容接口 (/v1/chat/completions)  │
│  ├── models.py    - 模型管理接口                            │
│  └── admin.py     - 管理接口                                │
├─────────────────────────────────────────────────────────────┤
│  服务层 (server/services/)                                   │
│  ├── backend_manager.py  - 后端管理器（管理模型加载/卸载）  │
│  ├── idle_monitor.py     - 空闲监控（自动卸载空闲模型）     │
│  └── inference_monitor.py - 推理监控                        │
├─────────────────────────────────────────────────────────────┤
│  后端层 (server/backends/)                                   │
│  ├── llama_backend.py    - llama.cpp 后端实现               │
│  └── backend_factory.py  - 后端工厂                         │
├─────────────────────────────────────────────────────────────┤
│  核心层 (server/core/)                                       │
│  ├── backend.py          - 后端抽象基类                     │
│  └── manager.py          - 模型配置管理器                   │
└─────────────────────────────────────────────────────────────┘
```

## 2. 关键组件分析

### 2.1 main.py 启动流程

1. 解析命令行参数 `--port`（默认 38520）
2. 初始化 ModelManager（加载 configs/models.yaml）
3. 初始化 BackendManager（管理模型生命周期）
4. 启动 uvicorn 服务器监听指定端口

### 2.2 BackendManager 职责

- 管理已加载的模型字典 `_loaded_models`
- 每个模型对应一个 LlamaBackend 实例
- 通过 `load_model()` 启动 llama-server 进程
- 通过 `chat/chat_stream()` 转发请求到对应的 llama-server

### 2.3 LlamaBackend 职责

- 启动 llama-server 子进程（端口默认 38521）
- 通过 HTTP 请求与 llama-server 通信
- 提供 `chat()` 和 `chat_stream()` 方法

## 3. 当前端口映射关系

当前架构中：
- **API 服务器端口**：由 `main.py --port` 指定（如 38520、39520）
- **llama-server 端口**：由模型配置中的 `port` 字段决定（默认 38521）

## 4. 数据流向

```
用户请求 → API层 (FastAPI) → BackendManager → LlamaBackend → llama-server
                                    ↓
                              ModelManager (模型配置)
```

## 5. 配置文件

- `configs/models.yaml` - 模型注册表
- `configs/server.yaml` - 服务器配置
- `configs/runtimes.yaml` - 运行时配置

## 6. 关键类说明

| 类名 | 文件路径 | 职责 |
|------|----------|------|
| BackendManager | server/services/backend_manager.py | 管理模型生命周期 |
| LlamaBackend | server/backends/llama_backend.py | llama.cpp 后端实现 |
| ModelManager | server/core/manager.py | 模型配置管理 |
| InferenceBackend | server/core/backend.py | 后端抽象基类 |
| IdleMonitor | server/services/idle_monitor.py | 空闲监控 |

## 7. 当前限制

当前一个 API 服务器实例只能管理一个 llama-server 端口，无法实现：
- 38520 API → 38521 llamaserver
- 39520 API → 39521 llamaserver

需要创建端口映射配置来实现多实例架构。
