# LLM 推理平台完整项目架构分析

## 1. 项目整体结构

```
c:\AI\LLM/
├── configs/                    # 配置文件目录
```

## 2. 后端架构详解 (server/)

### 2.1 API 层 (server/api/v1/)

| 文件 | 功能 | 路由前缀 | 主要端点 |
|------|------|----------|----------|
| **chat.py** | 聊天接口 | `/api/v1/chat` | POST /completions |
| **openai.py** | OpenAI 兼容接口 | `/v1` | GET /models, POST /chat/completions, POST /embeddings, POST /batches, POST /files |
| **anthropic.py** | Anthropic Claude 兼容接口 | `/v1` | POST /messages |
| **models.py** | 模型管理接口 | `/api/v1/models` | GET /, GET /types, GET /{model_id}, POST /{model_id}/load, POST /{model_id}/unload |
| **admin.py** | 系统管理接口 | `/api/v1/admin` | GET /system/stats, GET /backend/status, POST /backend/control |
| **logs.py** | 日志接口 | `/api/v1/logs` | POST /frontend, GET /frontend, DELETE /frontend |

**路由注册关系 (main.py):**
```python
app.include_router(models.router, prefix="/api/v1")
app.include_router(chat.router, prefix="/api/v1")
app.include_router(logs.router, prefix="/api/v1")
app.include_router(admin.router, prefix="/api/v1")
app.include_router(openai.router, prefix="/v1")      # OpenAI 兼容
app.include_router(anthropic.router, prefix="/v1")   # Anthropic 兼容
```

### 2.2 服务层 (server/services/)

| 文件 | 职责 | 关键功能 |
|------|------|----------|
| **backend_manager.py** | 后端管理器 | 管理模型加载/卸载、模型生命周期、多模型切换 |
| **idle_monitor.py** | 空闲监控 | 自动检测空闲模型并卸载、端口级监控 |
| **inference_monitor.py** | 推理监控 | Token统计、生成速度、请求统计 |
| **assistant_manager.py** | 助手管理 | OpenAI Assistant API 实现 |
| **batch_manager.py** | 批处理管理 | 批量请求处理 |
| **file_storage.py** | 文件存储 | 文件上传/下载管理 |

### 2.3 后端实现层 (server/backends/)

| 文件 | 职责 |
|------|------|
| **llama_backend.py** | llama.cpp 后端实现，启动 llama-server 子进程 |
| **sglang_backend.py** | SGLang 后端实现 |
| **embedding_backend.py** | Embedding 模型后端 |
| **backend_factory.py** | 后端工厂，根据配置创建对应后端 |

### 2.4 核心层 (server/core/)

| 文件 | 职责 |
|------|------|
| **backend.py** | 后端抽象基类 (InferenceBackend)，定义统一接口 |
| **manager.py** | ModelManager，管理模型配置加载和查询 |

### 2.5 适配器层 (server/adapters/)

| 文件 | 职责 |
|------|------|
| **anthropic_adapter.py** | Anthropic API 与 OpenAI API 之间的请求/响应转换 |
| **model_mapping.py** | 模型名称映射 |

## 4. 数据流向

### 4.1 聊天请求流程

```
用户输入 → ChatModern.vue → chat.js(store) → client.js(api) 
    → FastAPI路由 (chat.py/openai.py) → BackendManager 
    → LlamaBackend → llama-server (子进程)
```

### 4.2 模型加载流程

```
Models.vue → models.js(store) → client.js → models.py API
    → BackendManager.load_model() → LlamaBackend.initialize()
    → 启动 llama-server 子进程 → 等待健康检查
```

### 4.3 Anthropic 请求转换流程

```
Anthropic格式请求 → anthropic.py → AnthropicAdapter.convert_request()
    → OpenAI格式 → BackendManager.chat() → 返回后转换回Anthropic格式
```

## 6. 端口映射关系

### 6.1 当前架构

| 服务 | 端口 | 说明 |
|------|------|------|
| API 服务器 | 38520 (默认) | main.py --port 指定 |
| llama-server | 38521 (默认) | 模型配置中的 port 字段 |
| 前端开发服务器 | 38522 | vite.config.js 配置 |

### 6.2 请求转发链

```
浏览器:38522 → Vite代理 → API服务器:38520 → BackendManager → llama-server:38521
```

## 7. 配置文件说明

### 7.1 models.yaml
- 模型注册表
- 每个模型包含: id, path, type, backend, max_context, gpu_layers 等
- 支持 language-model, vision-language-model, reasoning-model, embedding-model

### 7.2 server.yaml
- 服务器基础配置
- CORS 设置
- 认证配置
- 速率限制

### 7.3 runtimes.yaml
- 运行时配置
- llama.cpp, SGLang 等后端的路径配置

## 8. 启动流程

### 8.1 后端启动 (main.py)

1. 解析命令行参数 `--port`
2. 初始化 ModelManager (加载 models.yaml)
3. 初始化 BackendManager
4. 注册 API 路由
5. 启动 uvicorn 服务器

### 8.2 前端启动

```bash
cd web
npm install
npm run dev  # 启动 Vite 开发服务器 (端口 38522)
```

## 9. 当前限制与问题

1. **单 llama-server 限制**: 一个 API 服务器实例只能管理一个 llama-server 端口
2. **端口映射**: 无法实现 38520→38521, 39520→39521 的多实例映射
3. **模型切换**: 切换模型时需要卸载旧模型，加载新模型
4. **内存限制**: 同时只能加载有限数量的模型 (默认 max_loaded_models=3)

## 10. 扩展点

如需实现多实例架构 (38520→38521, 39520→39521):

1. 创建端口映射配置 (configs/port_mapping.yaml)
2. 修改 BackendManager 支持根据 API 端口选择 llamaserver 端口
3. 创建多实例启动脚本 (start_multi.py)
4. 前端支持多后端切换
